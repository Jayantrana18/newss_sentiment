# News Sentiment vs Market Price Analysis Pipeline

An end-to-end data pipeline to analyze the relationship between financial news sentiment and stock market prices. Specifically designed to analyze **Reliance Industries (RELIANCE.NS)** using historical stock data and partner news headlines.

The pipeline comprises several stages: downloading data from AWS S3, fetching historical stock prices via Yahoo Finance, running sentiment analysis using a specialized financial NLP model (`ProsusAI/finbert`), and aggregating data using PySpark for correlation analysis between daily news sentiment and stock returns.

## 🚀 Features

- **Data Ingestion (AWS S3):** Securely download proprietary or large datasets (e.g., news headlines) from AWS S3 using `boto3`.
- **Market Data Retrieval:** Programmatically download daily historical stock prices (Open, High, Low, Close, Volume) using the `yfinance` API.
- **NLP Sentiment Analysis:** Leverage Hugging Face's `ProsusAI/finbert` (a pre-trained BERT model customized for financial text) to determine sentiment labels (Positive, Negative, Neutral) and confidence scores of news headlines.
- **Big Data Processing (PySpark):** Utilize PySpark to handle, clean, and aggregate daily sentiment data along with the computation of stock returns and lag features. 
- **Database Integration (MySQL):** Load raw, processed, and analyzed datasets into a MySQL database via PyMySQL and SQLAlchemy for further querying or dashboarding.
- **Statistical Correlation Analysis:** Compute Pearson correlation metrics between daily average sentiment scores and standard/next-day daily stock returns.

## � Key Insights & Findings

Based on the statistical aggregation generated by this pipeline, the **Pearson correlation between daily news sentiment and stock price returns is relatively low**. 
This quantitative finding suggests that while day-to-day fluctuations in stock prices do exhibit *some* dependence on media sentiment, broader market forces, technical indicators, and institutional trading factors play a much more dominant role in driving price action. Financial news serves as a supplementary signal rather than the primary predictor of short-term market movements over the evaluated period.

## 📋 Prerequisites & Installation

1. **Python:** Python 3.8+ is recommended. 
2. **Setup virtual environment (optional but recommended):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
3. **Install Dependencies:**
   Ensure you have all dependencies loaded, including database adapters:
   ```bash
   pip install -r requirements.txt
   pip install sqlalchemy pymysql
   ```
4. **Environment Variables (.env):** 
   Create a `.env` file in the root directory and ensure the following AWS configuration is set (if loading data from S3):
   ```env
   AWS_ACCESS_KEY_ID=your_access_key
   AWS_SECRET_ACCESS_KEY=your_secret_key
   AWS_REGION=us-east-1
   S3_BUCKET=your_s3_bucket_name
   ```

## ⚙️ How to Run

You can run the modules step-by-step:

### 1. Download News Data from S3
Downloads the raw financial news from the configured AWS S3 bucket to `data/`.
```bash
python -m src.s3_loader
```

### 2. Download Market Data
Downloads historical data for `RELIANCE.NS` (2012–2025) and saves it to `data/reliance_prices.csv`.
```bash
python -m src.download_reliance_stock_price
```

### 3. Compute Sentiment Analysis
Loads the FinBERT model to compute sentiment on up to 5,000 recent headlines. (Warning: This may take time if running entirely on CPU, but is typically quick for small batches). Results are saved to `data/news_with_sentiment.csv`.
```bash
python -m src.sentiment
```

### 4. Run PySpark Aggregation & Analytics
Aggregates the daily sentiments and merges them with daily stock returns. Outputs rolling correlations and a finalized dataset to `data/final_sentiment_price.csv`.
```bash
python -m src.spark_analysis
```

### 5. Load Data to MySQL (Optional)
If you have a local MySQL instance configured (database: `news_sentiment_db`), you can load all the raw and processed datasets directly into database tables for easy downstream querying:
```bash
python -m src.DB.load_csv_to_mysql
```

## 📉 End-to-End Orchestration (Optional)

Alternatively, the root `run_pipeline.py` script serves as a foundational wrapper outline for executing the entire pipeline holistically. It can be extended and customized to bind the individual scripts in `src/` sequentially.
